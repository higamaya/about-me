Languages

プログラミング言語は何が使えるの？というのが最も関心を集めるポイントかと思いますので、まずは各種プログラミング言語の習熟度をまとめてみました。

習熟度は5段階評価で、3以上は実務経験があるもの、2以下は実務経験はないけど触ったことがあるもの、というふうに区切ってあります。下記の図・表の赤いラインがその境界です。

習熟度実務経験評価の目安
5checkマスター
4check自由に駆使できる
3check実務で使える
2－それなりに使える
1－軽く触ったことがある

Java は、実務経験が10年と一番長く、もっとも精通している言語です。（※5年のブランクを埋めるために最新仕様をキャッチアップ中です。）

C++ は、もっとも好きな言語です。仮想関数テーブルの仕組みを語れる程度にはマスターしました。学生時代に最初にマスターした言語なので、かなり思い入れがあります。Javaに次いで得意です。

Additional Information

私の強み

作業工程としては上流から下流まで、システム担当範囲もフロントエンドからバックエンドまで幅広く経験しており、守備範囲が広いのが強みです。

新たな技術への挑戦

いま現在は、Java の最新仕様のキャッチアップなど、5年のブランクを埋めるので精一杯というのが正直なところですが、将来的には、未経験の言語やツールにも積極的にチャレンジして、自分のスキルセットをアップデートしていきたいと考えています。

直近の計画としては、C++ に代わる言語として Rust を習得する予定です。

“ソフトウェア開発者” に込めた想い

私は意図的に “ソフトウェア開発者” と名乗っています。これには、単純にプログラミングが大好きで、生涯プログラマーでいたい、という気持ちが含まれています。

もちろん、ただコードを書くだけの人になるつもりはなく、専門知識を駆使して顧客の課題を解決する技術者としての視点は大事にしたい。そういう立ち位置だと、“ITエンジニア” とか “システムエンジニア” なんて呼び方が一般的だと思いますが、“エンジニア” って名乗るのは自分には高尚すぎて少々照れくさい。かといって “プログラマー” では狭すぎる。

そんなこんなで色々と考えた結果、『プログラミングスキルを活かして人の役に立つもの（ソフトウェア）を作りたい（開発したい）』という想いを込めて “ソフトウェア開発者” と名乗ることにしました。

About This Site

CSSフレームワーク

本サイトのUIデザインは、Materialize というCSSフレームワークを使用して作成しました。

Materialize を選んだ理由は以下の2つです。

新しい挑戦

CSSフレームワークを使ってみたかった。Bootstrap など、CSSフレームワークの存在は知っていたが使ったことがなかった。

マテリアルデザイン

Materialize は、マテリアルデザインに基づくCSSフレームワークとして最も人気がある。The State of CSS 2020 では、前年から2つ順位を上げて5位にランクインしている。

ソースコードは、Materialize 公式から提供されているスターターテンプレートのひとつ Parallax Template をベースにしています。

制作にあたり、Materialize の様々な機能を試すと同時に、シンプルで分かりやすいデザインを心がけました。

ホスティング環境

本サイトは GitHub Pages でホスティングしています。GitHub のリポジトリから直接ソースを取得して公開してくれるので、デプロイ関連の設定作業を省くことができます。

ドメインは、デフォルトの github.io は使わず、カスタムドメインを使用しています。また、将来的に GitHub Pages で公開するサイトが増える可能性を考慮して、Apexドメインではなくサブドメインとしました。サイトごとに異なるサブドメインを割り当てるようにすることで、Apexドメインは1つだけで済みますし、さらにそのApexドメインを GitHub Pages 以外のホスティングサービスで利用することが可能となるからです。
